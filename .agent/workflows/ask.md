# Ask Workflow

This workflow is for answering general questions, whether they relate to the project or not.

## When to Use

Use this workflow when the user asks:

- General programming questions
- Technology or framework questions
- Best practices or design patterns
- Conceptual questions
- Questions about tools or libraries
- Questions unrelated to the project

## Workflow Steps

1. **Understand Question**
   - Read the question carefully
   - Identify the topic or domain
   - Determine if it relates to the project

2. **Gather Context**
   - Check if project context is relevant
   - Review related documentation if needed
   - Consider the user's level of understanding

3. **Formulate Answer**
   - Provide clear, accurate information
   - Use examples if helpful
   - Reference documentation or resources
   - Relate to project context if applicable

4. **Provide Additional Resources**
   - Suggest relevant documentation
   - Link to helpful resources
   - Recommend further reading if appropriate

## Response Format

Respond in Vietnamese with:

- Direct answer to the question
- Clear explanations
- Examples when helpful
- References to documentation
- No emojis

## Important Constraints

**INFORMATION ONLY**

- Do NOT modify any code
- Do NOT implement features
- Do NOT fix bugs
- ONLY provide information and answers
- Focus on knowledge sharing

## Important Rules

- Always respond in Vietnamese
- Use technical English terms appropriately
- Be accurate and factual
- Provide context and examples
- Keep answers focused and relevant
- Admit if you don't know something
- Suggest where to find more information
