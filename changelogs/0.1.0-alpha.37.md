# Changelog 0.1.0-alpha.37

## Features

- **UI Architecture**: Finalized migration to Feature-First modular architecture for better maintainability.
- **Agent Mentions**: Enhanced display of mentioned agents using modern badges/chips in chat messages.
- **Model Selection**: Added automatic selection of workspace default models or first available models in ChatInput.
- **Workflow Improvements**: Added automatic tab switching to "Installed" after successful installation of MCP servers, prompts, or agents.
- **Visuals**: Introduced persistent page animations, ambient glow, and brand marquee for a more dynamic landing page experience.

## Fixes

- **App Behavior**: Fixed an issue where the application would return to chat instead of closing when clicking the close button in settings.
- **Workspace Sync**: Resolved a bug where the chat screen didn't update correctly when switching workspaces.
- **Auto-rename**: Debugged and improved the chat auto-renaming service reliability.
- **Runtime Installation**: Fixed Windows-specific issue where a console window would appear during Python/Node.js runtime installation.
- **File System**: Separated AppData folders between development and production environments to prevent data conflicts.
- **Mermaid**: Improved Mermaid diagram caching and resolved file permission errors.

## Improvements

- **ChatInput**: Redesigned ChatInput to be more compact and balanced across different screen sizes.
- **Performance**: Optimized backend services by delegating logic to feature-specific modules.
- **I18n**: Updated Vietnamese translations and refined UI text for better clarity.
