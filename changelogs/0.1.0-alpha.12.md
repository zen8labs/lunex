# Changelog - 0.1.0-alpha.12

## Features

### Sentry Integration

- **Complete Sentry error tracking and performance monitoring integration**
  - Frontend (React): Automatic error boundary integration, session replay, performance monitoring
  - Backend (Rust): Automatic panic handling, error reporting, transaction tracking
  - Component performance tracking for critical UI components
  - Redux middleware for action tracking
  - LLM API call performance tracking
  - MCP tool execution tracking
  - Workspace and chat context tracking

### Build System

- **Sentry DSN embedded at build time**
  - Frontend: DSN embedded in bundle via Vite env vars
  - Backend: DSN embedded in binary via build.rs
  - GitHub Actions workflow updated to use repository variables
  - Desktop app tracking works out-of-the-box without runtime configuration

## Improvements

### Performance Monitoring

- Added performance tracking hooks for critical components:
  - `MessageItem` (30ms threshold)
  - `ChatMessages` (50ms threshold)
  - `MessageList` (100ms threshold)
  - `MarkdownContent` (200ms threshold)
  - `ChatInput` (50ms threshold)
  - `ChatSidebar` (100ms threshold)

### Error Tracking

- Comprehensive error tracking with context:
  - User actions tracking
  - API call performance monitoring
  - Streaming performance metrics
  - Tool execution tracking
  - Database operation monitoring (ready for use)

### Developer Experience

- Sentry helper utilities for consistent tracking patterns
- Automatic data sanitization (API keys, passwords, tokens)
- Smart filtering to reduce noise
- Production-only tracking by default

## Technical Details

### Dependencies

- Added `@sentry/react@10.32.1` for frontend
- Added `@sentry/vite-plugin@4.6.1` for source maps
- Added `sentry@0.34` for backend with features: backtrace, contexts, panic, anyhow, debug-images
- Added `tracing@0.1` and `tracing-subscriber@0.3` for structured logging

### Files Changed

- Frontend: `src/main.tsx`, `src/ui/atoms/ErrorBoundary.tsx`, `src/store/sentryMiddleware.ts`, `src/lib/sentry-utils.ts`
- Backend: `src-tauri/src/lib.rs`, `src-tauri/src/lib/sentry_helpers.rs`, `src-tauri/build.rs`
- Components: Added performance tracking to 6 critical components
- Workflow: Updated `.github/workflows/build.yaml` for Sentry DSN integration

## Notes

- Sentry is disabled in development mode by default
- Set `SENTRY_ENABLED=true` to test locally
- DSN must be set in GitHub repository variables for CI/CD builds
- All sensitive data is automatically redacted before sending to Sentry
