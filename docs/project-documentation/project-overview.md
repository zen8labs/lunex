# Nexo Project Overview

**Type:** Desktop Application (Monolith)  
**Primary Language:** TypeScript (Frontend), Rust (Backend)  
**Architecture:** Local-first, Hybrid (React + Tauri + Rust)  
**Last Updated:** 2025-01-27

## Executive Summary

Nexo is a powerful, cross-platform desktop AI assistant built with Tauri, React, and Rust. It provides a seamless interface for interacting with multiple LLM providers while offering advanced features like MCP (Model Context Protocol) integration, workspace management, and extensible tool support.

### Key Characteristics

- **Local-First Architecture**: All conversations stored locally in SQLite, no data sent to third parties
- **Multi-LLM Support**: Connect to multiple LLM providers (OpenAI, Anthropic, Google, and more)
- **MCP Integration**: Full support for Model Context Protocol servers
- **Workspace Management**: Organize conversations into workspaces with workspace-specific settings
- **Extensible**: Python and Node.js runtime support for executing tools
- **Privacy-Focused**: No telemetry or data collection

## Architecture Pattern

Nexo follows a **layered architecture** with clear separation between:

1. **Frontend Layer** (React/TypeScript): UI components, state management, user interactions
2. **IPC Bridge** (Tauri): Inter-process communication between frontend and backend
3. **Backend Layer** (Rust):
   - **Commands Layer**: Tauri command handlers (API endpoints)
   - **Services Layer**: Business logic and orchestration
   - **Repository Layer**: Data access and persistence
   - **Models Layer**: Data structures and types

## Getting Started

### Prerequisites

- Node.js (LTS version)
- Yarn package manager
- Rust (latest stable version)
- Platform-specific dependencies (see README.md)

### Quick Start

```bash
# Install dependencies
yarn install

# Generate TypeScript bindings
yarn gen:bindings

# Run in development mode
yarn tauri:dev

# Build for production
yarn tauri:build
```
